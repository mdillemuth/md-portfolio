The Challenge

Build a full-stack web app where users can sign up and create a list of their favorite movies.

The Goal

Write readable code that is both easy to understand and maintain.

The Process

  1. Start by building the database and write the back-end logic of how the app can be accessed.
  2. Develop the interactive front-end that enables users to experience the app's features.
  3. Use version control along the way to track changes and host the code on GitHub.

Application Structure

Back-End

  • Using Node and Express, design a RESTful API and testing the endpoints with Postman.
  • Using MongoDB and Mongoose, create a non-relational database and business logic.
  • Using Passport and JWT, authenticate and authorize users of the app.
  • Using Heroku, host the app’s back-end API.

Front-End

  • Using React, create separate components for the different views of the application.
  • Using React-Router-Dom, implement logic to handle app requests and navigate users to different views within the app.
  • Using Redux and React-Redux, manage the app’s state and data flow by implementing the flux design pattern.
  • Using Netlify, host the app’s front-end client.

Component

The alert component is used to alert the user of successful or unsuccessfuls account login, registration, or update attempts. It's integration in the app is very concise, meaning that it does not have to be added to a component that wants to use it.

Implementation

The component is rendered directly from the top parent component and is displayed under the navigation bar. This means that components that want to call the Alert do not need to import it directly, they only need to create the action for it with Redux.

Redux is used to manage the Alert state. A timeout function has been added to make alerts automatically disappear after a fixed time. This was implemented in order to improve UX by not requiring users to have to click to remove alerts themselves.

Component

The loading spinner component is used to inform the user that the app is calling the backend's API and is handling their request. During development, it was noticed that login attempts could take over 5 seconds and it begged the question, "Is the app doing anything?". To answer that question, a simple loading spinner was implemented.

Implementation

The spinner gif was created from a free online service, and was customized to use the same color red of the app's theme. Components that use the loading gif import and render it directly.

In the Redux store for 'users', the loading state is manipulated on actions that require an API call. The loading state is only true after a API request has been sent and before it has been resolved. The simplicity of its implementation reinforces the readability of the app's code.

Component

A key feature of the app is the user's ability to filter and sort movies. Users can filter by a search query, by director, or genre. They are able to sort by movie title alphabetically and sorting is applied to the filtered set of data.

Implementation

The filter array method in JavaScript is used in order to filter by director, genre, or query. Capitalization and leading or trailing spaces are accounted for in the queries. The 'orderBy' function from the library, Lodash, is used in order to perform sorting alphabetical sorting by movie title.

Redux is used to handle the movies state and facilitates persisting the filters or sorting methods applied to the data set. The actions to sort or filter movies are able to be called the UI component that handles the filtering and sorting.

Personal Reflection

This project has unequivocally been the defining learning experience of my web development education. React and Redux were completely new technologies for me and I felt humbled by their complexity. What I came to learn however, was that these libraries are incredibly powerful and capable of scaling extremely well.

Overcoming Obstacles

My first challenge was adopting the new paradigm of viewing an app as a collection of modules that are all interacting with eachother. This required me to modify my perspective and develop a more complete understanding of web development as a whole.

Before implementing Redux, there was one area of the app that was especially trying for me. The issue I was having was that I was unable to refresh the page of a component that was rendering data fetched from the backend by the API.

The component was able to render the first time because the data was being passed as a prop. However, on refresh, the app would attempt to render the component before the data could be fetched from the api. Then the app would throw an error stating that it could not filter an undefined data set.

The solution I implemented was to use a ternary operator in the component's return statement. This enabled the component to check that the data was in the state before rendering the elements that required said data (and before leveraging Redux's state management capabilities).

Realizing Success

As mentioned, one of my goals for this project was to write clean code. React and Redux were great tools to accomplish this.

With Redux, I decided to use the @reduxjs/toolkit library in order to create 'slices' for each state being managed by Redux. This enabled me to consolidate the state, reducers, and actions into a single file. These files were concise and focused on a single aspect of the app being handled with Redux, namely users, movies, and alerts.

Finally, the design and theme were a pleasure to implement. I was able to leverage my knowledge of UI design in order to create a clean and visually-appealing layout in my app. I improved the UX by implementing features such as the loading spinner, alert messages, and navigating the user back to the login screen after logging out.
